## Профиль
- Роль: координатор креативов — принимает готовые креативы от дизайнеров, переименовывает, заливает на рекламные сети
- Работает с 8+ креативщиками
- Заливает на: Unity Ads, YASU, Google Ads, Meta и другие сети
- Получает задачи от менеджеров (какой креатив на какую сеть)
- Аналитика перформанса креативов ведётся в Superset (смотрят Zoriana Omelchuk (CMO) и менеджеры)

## Текущий процесс

```
Дизайнер → [файл с произвольным именем]
         → Hanna переименовывает по конвенции
         → загружает на Google Drive (папки по играм)
         → менеджер говорит, куда залить (сеть + библиотека/кампания)
         → Hanna заливает вручную на сети
         → данные забираются из сетей по API → Superset
```

**Важный нюанс:** креативы запускаются пачками (например, 3 шт. одновременно для A/B-теста), даже если дизайнеры сдают их в разные дни. Hanna ждёт, пока все готовы, и заливает разом.

## Нейминговая конвенция (текущая)

### Правила
1. Только латиница (была проблема с украинской "і" vs английская "i")
2. Нижнее подчёркивание как разделитель (без пробелов)
3. Без кириллицы

### Форматы

| Тип | Префикс | Формат | Пример |
|-----|---------|--------|--------|
| Видео (реклама) | V | `V{NNN}_{код_1С}_{название}_{размер}_{длительность}` | `V003_7448_CarCrash_1920x1080_30s` |
| Видео (стор) | SV | `SV{NNN}_{код_1С}_{название}_{размер}_{длительность}` | `SV001_7448_CarCrash_1920x1080_30s` |
| Баннер | B | `B{NNN}_{код_1С}_{название}_{размер}` | `B011_7448_CarCrash_1020x500` |
| Playable | PLAY | `PLAY_{NNN}_{код_1С}_{название}_{сеть}` | `PLAY_001_7448_CarCrash_Unity` |

### Сезонные маркеры
- Раньше: суффикс `_new_year` в разных местах (непоследовательно)
- Сейчас: суффикс-код `UE` сразу после номера, без подчёркивания: `V020UE_...`

## Ключевые боли

1. **Ручное переименование** — Hanna вручную переименовывает каждый файл от дизайнеров по конвенции. _"Дизайнеры отдают креативы, я их переименовываю."_

2. **Историческая неконсистентность нейминга** — старые креативы названы по-разному: без кода 1С, с кириллицей, сезонные теги в разных позициях. Аналитика ломается.

3. **Нет превью в каталоге** — при просмотре списка нет визуального представления. _"Зоряна говорила, что неплохо было бы иметь превьюшки."_

4. **Ручная заливка на сети** — для каждого креатива Hanna вручную загружает файлы в библиотеки/кампании на каждой сети.

5. **Нет единой базы** — креативы живут в папках на диске. Нет поиска, нет статусов, нет связи с аналитикой.

## Инсайты для продукта

### Что подтвердилось
- Креативы — важный объект аналитики для UA (подтверждает US-Pub-11, US-PubC-06)
- Нейминг критичен для связки с API сетей и отображения в аналитике
- Ручной труд координатора — бутылочное горлышко

### Что нового узнали
- **Процесс батчевый:** креативы запускаются пачками для A/B-тестирования, даже если создаются в разные дни
- **Playables — отдельный канал:** часть через PlayTurbo/Mintegral, не хранятся в общих папках
- **Код 1С — ключевой идентификатор:** связывает креатив с игрой в учётной системе
- **Концепт (порядковый номер) — единица творческой идеи:** менеджеры обсуждают "21-й креатив сработал, сделайте похожий"
- **Размер кадра = адаптация под сеть**, а не новый креатив

### Что влияет на архитектуру
- Сущность Creative = концепт (идея). Варианты размеров = рендиции одного концепта
- Нужна связь: Creative → Game (через код 1С) → Campaign → Network → Performance (из Superset)
- Старые креативы нельзя переименовать в сетях — только маппинг в базе
