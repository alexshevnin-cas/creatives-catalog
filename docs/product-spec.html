1. Проблема

Рекламные креативы (видео, баннеры, playables) управляются вручную: дизайнеры сдают файлы → координатор переименовывает → заливает на рекламные сети. Нет единого каталога, нет автоматизации нейминга, нет связи с аналитикой.

Job-to-be-Done: «Олег Шлямович подходит и спрашивает "что по Vegas Crime, какие последние концепты, геймплейные или нет?" — и мне нужно ответить за 30 секунд, а не за 10 минут копания в папках.» — Zoriana

Последствия:

  • Аналитика в Superset ломается из-за неконсистентного нейминга
  • Координатор тратит время на ручное переименование и заливку
  • Менеджеры не видят превью креативов в контексте перформанса
  • Нет единого реестра: что создано, что запущено, что работает
  • Google Drive заканчивается (~100 ГБ + ~130 ГБ на подходе) — негде хранить файлы

2. Цели

  1. Единый каталог креативов с поиском, превью, метаданными и статусами — единое место, где видно что создано, в каком состоянии и куда залито
  2. Автоматический нейминг по конвенции — исключает человеческие ошибки и обеспечивает корректный проброс в аналитику (имя в каталоге = имя в Superset)
  3. Ускорение работы — удобный интерфейс с дропдаунами, автоопределением размеров, drag & drop и загрузкой на сервер вместо ручных операций с файлами и именами

4. Нейминговая конвенция

Правила

  1. Только латиница (ASCII)
  2. Разделитель: _ (нижнее подчёркивание)
  3. Без пробелов, без кириллицы
  4. Формат зависит от типа креатива
  5. Сеть НЕ включать в имя — источник определяется через MMP/API
  6. Размеры и длительность определяются автоматически из загруженного файла
  7. Уникальный код концепта (8 символов) добавляется в конце имени через _: 6 символов системного хэша + 2 символа направления (UA / CP / DS)
  8. Без ограничений на размер загружаемого файла

Имя концепта (без размеров)

ТипШаблонПример
VideoV{NNN}_{код_1С}_{shortname}_{uid}V003_7448_CarCrash_a1b2c3UA
BannerB{NNN}_{код_1С}_{shortname}_{uid}B011_7448_CarCrash_d4e5f6UA
PlayablePLAY_{NNN}_{код_1С}_{shortname}_{uid}PLAY_001_7448_CarCrash_g7h8i9UA

Где {uid} = 6 символов хэша + 2 символа направления:

  • UA — закупка (User Acquisition)
  • CP — кросс-промо (Cross-Promo)
  • DS — DSP

Имя рендиции (полное имя файла)

ТипШаблонПример
Video{concept}_{WxH}_{duration}sV003_7448_CarCrash_1920x1080_30s_a1b2c3UA
Banner{concept}_{WxH}B011_7448_CarCrash_1020x500_d4e5f6UA
Playable{concept}PLAY_001_7448_CarCrash_g7h8i9UA

Иерархия сущностей

Game (код 1С = 7448, "Car Crash Simulator")
 └── Creative Concept (V003 = 3-я идея, tags: gameplay, UGC)
      └── Rendition (1920x1080_30s, 1080x1920_30s, 1280x720_15s)
           └── Deployment (Meta, TikTok, Google Ads — V1.2+)

Concept — единица творческой идеи (один порядковый номер).
Rendition — адаптация под формат (разные разрешения/длительности одного концепта).
Deployment — факт заливки рендиции на конкретную сеть (V1.2+).

6. Целевой процесс

Сейчас

Дизайнер → файл с произвольным именем
         → Hanna переименовывает вручную
         → Google Drive (папки по играм)
         → Hanna вручную заливает на сети
         → API сетей → Superset (нейминг иногда ломается)

Целевой (V1.0): Каталог + хранилище на сервере

Дизайнер → Каталог (форма: [Игра ▼] [Тип ▼] + файл + теги)
         → Система автоименует + определяет размеры из файла
         → Файл загружается на сервер CAS
         → Thumbnail генерируется автоматически
         → Менеджер отмечает сети в каталоге
         → Координатор видит готовые файлы с правильными именами, заливает на сети
         → MMP → Superset (нейминг всегда корректный)

Целевой (V1.2): Автозаливка на сети

Координатор → выбирает креатив + сети + кампании
            → Кнопка "Залить" → API Meta / Google / TikTok / Unity
            → Менеджер на стороне сети выбирает по имени

Целевой (V2+): Аналитика по креативам

Каталог ← Performance data (spend, ROAS, конверсии)
       ← Retention по когорте, привлечённой креативом
       → CMO видит: какой креатив перформит, mislead или нет

7. UI каталога

Древовидная структура

▸ CarCrash (7448)
  ▸ Video                                         [12]  [+ Добавить]
    | #   | Теги      | Превью | Концепт              | Файлы     | Сети                    | Статус   |
    | 012 | gameplay  | [img]  | V012_7448_CarCrash   | 2 файла ▸ | [Mintegral] [FB] [...]  | Draft    |
    | 011 | UGC       | [img]  | V011_7448_CarCrash   | 3 файла ▸ | [Mintegral] [FB] [...]  | Active   |
    ...
                                                     └── V011_7448_CarCrash_1920x1080_30s  [Скачать]
                                                         V011_7448_CarCrash_1080x1920_30s  [Скачать]
                                                         V011_7448_CarCrash_1280x720_15s   [Скачать]
  ▸ Banner                                        [12]  [+ Добавить]
  ▸ Playable                                      [12]  [+ Добавить]

Принципы UI

  • Сортировка: новые концепты сверху (DESC по номеру)
  • Фильтры: игра, тип, теги, статус, поиск по имени
  • Теги — кликабельные бейджи (gameplay, mislead, UGC, seasonal и др.)
  • Сети — кликабельные бейджи (toggle on/off)
  • Статус — дропдаун inline
  • Файлы — раскрывающийся список рендиций
  • Кнопка "+ Добавить" — модалка с предзаполненными игрой и типом
  • Превью — thumbnails (генерируются на сервере)

Форма добавления (модалка)

  • Два дропдауна: Игра, Тип (предзаполнены если открыто из каталога)
  • Мульти-селект тегов
  • Drag & drop файла или импорт по URL Google Drive
  • Размеры и длительность определяются из файла автоматически
  • Live-preview сгенерированного имени
  • Кнопка "Добавить"

8. Хранилище файлов

Решение: собственный сервер

Google Drive заканчивается (~100 ГБ занято + ~130 ГБ от двух крупных игр на подходе). Файлы хостятся на серверах CAS.

Открытый вопрос: стоимость 1 ТБ на сервере vs Google Drive Business — узнать у Вовы.

Структура хранения

/storage/creatives/
├── 7448_CarCrash/
│   ├── Video/
│   │   ├── V001_7448_CarCrash_1920x1080_30s.mp4
│   │   ├── V001_7448_CarCrash_1080x1920_30s.mp4
│   │   └── ...
│   ├── Banner/
│   └── Playable/
├── thumbnails/
│   └── ... (авто-генерация)

Thumbnails

Генерируются на сервере при загрузке: Pillow для изображений, ffmpeg для видео (первый кадр).

9. Фазы

V1.0 — Хранилище + каталог (заменяет Google Drive)

  • ☑ Древовидный каталог: Игра → Тип → Концепт → Рендиции (прототип)
  • ☑ Автонейминг по конвенции (прототип)
  • ☑ Авто-определение размеров и длительности из файла (прототип)
  • ☑ Фильтры, поиск, статусы (прототип)
  • ☑ Отметка сетей (Mintegral, FB, TikTok, Google Ads) (прототип)
  • ☐ Хранилище на сервере CAS (upload + thumbnails)
  • ☐ Теги (gameplay, mislead, UGC, seasonal и др.)
  • ☐ PostgreSQL вместо SQLite
  • ☐ Справочник игр (код 1С + short_name)
  • ☐ Аутентификация

V1.2 — Автозаливка на сети через API

  • Кнопка «Залить на Meta / Google / TikTok / Unity»
  • API интеграция с рекламными сетями (у Vitalii Tereshchenko есть креды и аккаунты)
  • YouTube: заливка видео по API → менеджер выбирает по имени в кампании
  • Meta: каталог ассетов через API

V2+ — Аналитика по креативам

  • Маркетинговые метрики: Impression → Click → Install конверсии, Spend vs ROAS
  • Кросс-креативные сравнения: mislead vs non-mislead (конверсия высокая, retention нулевой = mislead)
  • Продуктовые метрики на креатив: Retention по когорте, привлечённой конкретным креативом
  • Требуется: behavior events в SDK + атрибуция (MMP) + сведение в аналитике
  • Маппинг старых креативов
  • Интеграция с 1С для автосинхронизации справочника игр

11. Открытые вопросы

  1. Каталог или бот? → Каталог с хранилищем на сервере (решено)
  2. Лимиты длины имени по рекламным сетям (Meta, Google Ads, Unity, Mintegral, TikTok) — нужно исследование. Особенно актуально после добавления 8-символьного uid в конце
  3. Zoriana Omelchuk (CMO) → показан прототип, скоуп подтверждён (2026-03-19)
  4. Vitalii Tereshchenko (UA) — посмотреть его прототип каталога, переиспользовать что можно. У него есть креды сетей
  5. PlayTurbo/Mintegral playables — загружаются отдельным путём, как учитывать?
  6. Права доступа — кто может загружать, кто может менять статус?
  7. Стоимость хостинга — узнать у Вовы: 1 ТБ на сервере vs Google Drive Business
  8. Уникальный код в сетях — как uid попадает в рекламные сети? Не сломает ли матчинг в Superset? Не обрежут ли сети?
  9. Ориентация в рамках концепта — нужна ли группировка рендиций по вертикальной/горизонтальной ориентации для аналитики?
  10. Формализация short_name — нужны ли правила формирования? Сейчас выбирается произвольно
  11. iOS/Android — одна игра или разные? — У iOS и Android разные коды 1С. Является ли это одной сущностью Game или двумя?
  12. Медиафайл как отдельная сущность — нужно ли отвязывать файл от кода 1С для переиспользования между платформами? (предложение Kapatsyn + Максим Телига)
  13. Сжатие при загрузке — нужно ли? Сети вероятно сами пережимают, но уточнить ограничения на вход
  14. Правила архивации — что происходит со старыми файлами? Когда и как переносить в архив? Узнать текущие правила у Ани
