1. Проблема

Рекламные креативы (видео, баннеры, playables) управляются вручную: дизайнеры сдают файлы → координатор переименовывает → заливает на рекламные сети. Нет единого каталога, нет автоматизации нейминга, нет связи с аналитикой.

Последствия:

  • Аналитика в Superset ломается из-за неконсистентного нейминга
  • Координатор тратит время на ручное переименование и заливку
  • Менеджеры не видят превью креативов в контексте перформанса
  • Нет единого реестра: что создано, что запущено, что работает

2. Цели

  1. Единый каталог креативов с поиском, превью, метаданными и статусами — единое место, где видно что создано, в каком состоянии и куда залито
  2. Автоматический нейминг по конвенции — исключает человеческие ошибки и обеспечивает корректный проброс в аналитику (имя в каталоге = имя в Superset)
  3. Ускорение работы — удобный интерфейс с дропдаунами, автоопределением размеров, drag & drop и синхронизацией с Google Drive вместо ручных операций с файлами и именами

3. Пользователи

РольЧто делаетКак сейчас
Координатор (Hanna) Организует сбор креативов от команд (моушн, дизайн и др.), именует, диспетчеризирует по сетям, отслеживает статусы Ручной сбор файлов, переименование, заливка на сети через чаты и папки
CMO (Zoriana) Видит полную картину: все креативы в одном месте с превью, статусами и метаданными Разрозненные папки, чаты, Superset

Система помогает координатору: автоматизирует нейминг, собирает все креативы в одном месте с превью и статусами, синхронизирует с Google Drive.

4. Нейминговая конвенция

Правила

  1. Только латиница (ASCII)
  2. Разделитель: _ (нижнее подчёркивание)
  3. Без пробелов, без кириллицы
  4. Формат зависит от типа креатива
  5. Сеть НЕ включать в имя — источник определяется через MMP/API
  6. Сезонность — фиксированная позиция сразу после номера, без подчёркивания
  7. Размеры и длительность определяются автоматически из загруженного файла

Имя концепта (без размеров)

ТипШаблонПример
VideoV{NNN}{seasonal}_{код_1С}_{shortname}V003UE_7448_CarCrash
BannerB{NNN}{seasonal}_{код_1С}_{shortname}B011_7448_CarCrash
PlayablePLAY_{NNN}{seasonal}_{код_1С}_{shortname}PLAY_001_7448_CarCrash

Имя рендиции (полное имя файла)

ТипШаблонПример
Video{concept}_{WxH}_{duration}sV003UE_7448_CarCrash_1920x1080_30s
Banner{concept}_{WxH}B011_7448_CarCrash_1020x500
Playable{concept}PLAY_001_7448_CarCrash

Сезонные маркеры

КодЗначение
STDСтандартный (не отображается в имени)
UEUniversal Event (сезонное промо)
NYNew Year
EAEaster

Формат: суффикс сразу после номера без подчёркивания — V003UE_.... Для STD суффикс пустой — V003_....

Иерархия сущностей

Game (код 1С = 7448, "Car Crash Simulator")
 └── Creative Concept (V003 = 3-я идея)
      └── Rendition (1920x1080_30s, 1080x1920_30s, 1280x720_15s)

Concept — единица творческой идеи (один порядковый номер).
Rendition — адаптация под формат (разные разрешения/длительности одного концепта).

6. Целевой процесс

Сейчас

Дизайнер → файл с произвольным именем
         → Hanna переименовывает вручную
         → Google Drive (папки по играм)
         → Hanna вручную заливает на сети
         → API сетей → Superset (нейминг иногда ломается)

Целевой (V1): Каталог + Google Drive

Дизайнер → Каталог (форма: [Игра ▼] [Тип ▼] [Сезон ▼] + файл)
         → Система автоименует + определяет размеры из файла
         → Файл загружается на Google Drive в правильную папку
         → Thumbnail подтягивается из Drive API
         → Менеджер отмечает сети в каталоге
         → Координатор видит готовые файлы с правильными именами, заливает на сети
         → MMP → Superset (нейминг всегда корректный)

Целевой (V2): Автозаливка

Дизайнер → загружает в Catalog
         → Менеджер отмечает сети + кампании
         → Система автоматически заливает через API сетей
         → Performance data возвращается в Catalog

7. UI каталога

Древовидная структура

▸ CarCrash (7448)
  ▸ Video                                         [12]  [+ Добавить]
    | #   | Сезон | Превью | Концепт              | Файлы     | Сети                    | Статус   |
    | 012 | STD   | [img]  | V012_7448_CarCrash   | 2 файла ▸ | [Mintegral] [FB] [...]  | Draft    |
    | 011 | STD   | [img]  | V011_7448_CarCrash   | 3 файла ▸ | [Mintegral] [FB] [...]  | Active   |
    ...
                                                     └── V011_7448_CarCrash_1920x1080_30s  [Скачать]
                                                         V011_7448_CarCrash_1080x1920_30s  [Скачать]
                                                         V011_7448_CarCrash_1280x720_15s   [Скачать]
  ▸ Banner                                        [12]  [+ Добавить]
  ▸ Playable                                      [12]  [+ Добавить]

Принципы UI

  • Сортировка: новые концепты сверху (DESC по номеру)
  • Фильтры: игра, тип, сезон, статус, поиск по имени
  • Сети — кликабельные бейджи (toggle on/off)
  • Статус — дропдаун inline
  • Файлы — раскрывающийся список рендиций
  • Кнопка "+ Добавить" — модалка с предзаполненными игрой и типом
  • Превью — thumbnails из Google Drive API

Форма добавления (модалка)

  • Три дропдауна в одну строку: Игра, Тип, Сезонность (предзаполнены если открыто из каталога)
  • Drag & drop файла
  • Размеры и длительность определяются из файла автоматически
  • Live-preview сгенерированного имени
  • Кнопка "Добавить"

8. Google Drive интеграция

Хранилище

Google Drive — основное хранилище файлов. Не заменяем, а интегрируемся.

Структура папок

[Creatives]/
├── CarCrash (7448)/
│   ├── Video/
│   │   ├── V001_7448_CarCrash/
│   │   │   ├── V001_7448_CarCrash_1920x1080_30s.mp4
│   │   │   └── V001_7448_CarCrash_1080x1920_30s.mp4
│   │   └── V002UE_7448_CarCrash/
│   ├── Banner/
│   └── Playable/

Thumbnails

Google Drive API отдаёт thumbnailLink для видео и изображений — не нужен ffmpeg, не нужно хранить превью локально.

9. MVP Scope

V1 (в разработке)

  • ☑ Древовидный каталог: Игра → Тип → Концепт → Рендиции (прототип)
  • ☑ Автонейминг по конвенции (прототип)
  • ☑ Авто-определение размеров и длительности из файла (прототип)
  • ☑ Фильтры, поиск, статусы (прототип)
  • ☑ Отметка сетей (Mintegral, FB, TikTok, Google Ads) (прототип)
  • ☐ Google Drive интеграция (upload + thumbnails)
  • ☐ Google OAuth 2.0
  • ☐ PostgreSQL вместо SQLite
  • ☐ Справочник игр (код 1С + short_name)

V2+

  • Автозаливка на рекламные сети через API
  • Перформанс-данные из Superset в карточке креатива
  • A/B-группы с автоматическим запуском
  • Drive watcher (фоновый импорт файлов из Drive)
  • Маппинг старых креативов
  • Интеграция с 1С для автосинхронизации справочника игр
